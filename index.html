Базовые понятия цифровой инженерии знаний

Ревизия №2 от 2020.03.27

Мотивация

Смысл развития технологий автоматизации работы со знаниями в том, чтобы разгрузить восприятие человека, устранить необходимость ежедневно оперировать огромным объём относительно несложной структурированной информации, перестать тратить время на довольно тривиальный поиск и анализ информации для решения рутинных задач. С развитием информационных технологий кратно возросло количество информационного шума, который окружает каждый полезный артефакт в этом виртуальном мире. Такое ощущение, что в результате этого в современном мире люди оказались перегружены интеллектуально. Получается, что дальнейшее развитие информационных технологий хоть и даёт больше функциональных возможностей, но ещё больше перегружает восприятие, а значит обессиливает людей, лишает их радости жизни, возможности уделять внимание семье и самим себе.

С другой стороны стоит выживание. Быстрая трансформация и оптимизация всех процессов в обществе и индивидуальной жизни для решения стремительных и смертоносных проблем. Например, война, эпидемия или стихийное бедствие. Человечество копит огромный багаж знаний о том, как жить и выживать в этом мире, но не может им воспользоваться без затраты огромных усилий и времени. Я думаю, что вопрос эффективного оперирования практическими знаниями - это вопрос и самореализации, и выживания.

Задачи по автоматизации обработки данных и создания/копирования/передачи информации широко известны. Но без автоматизации хотя бы тривиальных способов работы со знаниями, они дают мало пользы. Мы уменьшаем нагрузку с одной стороны, но увеличеваем её с другой стороны. Я не знаю удастся ли замкнуть круг задач в устойчивую экосистему, но хочется попробовать сделать шаг вперёд от информационных технологий к инженерии знаний.

Цель

Найти способ кооперации людей и машин, направленный на наиболее эффективное и надёжное решение задач выживания, при котором остаётся максимум свободного времени и энергии на настоящую жизнь и развитие за пределами ограничений враждебной среды, в которой мы родились и живём.

Методика

В общем случае навряд ли можно измерить знания численно. Но если рассмотреть знания в контексте их конкретного цифрового представления, то можно выделить некоторые базовые структуры данных, которые могут быть использованы для описания знаний. Их конфигурация и размер вполне могут быть определны.

knit

По аналогии с bit для измерения информации, knit - это наименьшая единица измерения знаний для цифрового устройства. Если представить, что знания можно выразить совокупностью семантических сущностей, и каждую такую сущность обозначать через глобально уникальный идентификатор, то он и будет структурой данных для описания knit'а, а его размер - размером knit'а. Если использовать для идентфикации классический uuid v4. Тогда значения knit'ов могут выглядеть вот так:

ac2fb456-5936-48dc-934c-1bfcf6bd71e6
8a048d4c-93c0-45b8-8a36-e8a3cd80164d
71076d29-d4ff-430a-b9bd-9c9ee6e5584e

Таким образом размер knit'а равен

• 128 бит (16 байт) в бинарном представлении;
• 36 байт в текстовом представлении.

knyte

По аналогии с byte для измерения информации, knyte - это наименьшая единица измерения знаний, доступных для воспринятия человеком. Если byte - это количество bit, необходимых для кодирования одного символа в тексте, то knyte - это количество knit, необходимых для полного описания одной абстрактной семантической сущности.

Опишем абстрактную семантическую сущность следующей структурой данных:

core entity: entity id → knit → uuid v4
vector initial entity: entity id → knit → uuid v4
vector terminal entity: entity id → knit → uuid v4

Тогда размер knyte'а равен 3 knit'ам.

sizeof(knyte) === 3 * sizeof(knit) // 48 bytes

knyte web

Совокупность связанных между собой knyte'ов называется сетью knyte'ов. Мы исходим из предположения, что она может описывать абстрактными знания.

Что такое абстрактные знания? Это знания без конкретной интерпретации и применения. Некая обобщённая структура, выражающая суть такого явления как знания.

information map

Для интерпретации семантических сущностей и их сетей человеком или машиной, нужно иметь возможность привязывать к ним информацию. Информация - это интерпретируемые данные. Объём этих данных измеряется в байтах. Данные могут интерпретироваться как людьми (например, текстовая, визуальная, звуковая информация), так и алгоритмами (например, числовые, строковые, бинарные типы данных или составные структуры).

Если к абстрактной семантической сущности привязать информацию, то она становится обычной семантической сущностью. Её смысл которой можно интерпретировать в контексте связей с другими семантическими сущностями.

В качестве структуры данных для описания информационной части семантических сущностей используется ассоциативный массив (hashmap). Для набора из 3 семантических сущностей информационная карта может выглядеть так:

ac2fb456-5936-48dc-934c-1bfcf6bd71e6: "first entity" // 12 bytes
8a048d4c-93c0-45b8-8a36-e8a3cd80164d: "100500" // 6 bytes
71076d29-d4ff-430a-b9bd-9c9ee6e5584e: "" // 0 bytes

Объём данных в представленной информационной карте равен 18 байтам.

Каждый элемент в information map называется information record.

unified semantic web

Объединённые knyte cloud и её information map образуют унифицированную семантическую сеть (unified semantic web). Унифицированная, потому что состоит из однотипных структур данных: knyte + information record. Семантическая, потому что её элементами являются семантические сущности. Сеть, потому что её объекты могут быть произвольно связаны друг с другом и не имеют связей ни на какие внешние объекты. В таком виде семантическая сеть имеет 2 размерности:

• размер структуры в knyte'ах, который вычисляется по knyte web;
• объём информации в byte'ах, который вычисляется по information map.

Итого описанная в примерах семантическая сеть имеет размер 3 knytes x 18 bytes.

knowledge base

Конкретная программно-аппаратная реализация концепции унифицированной семантической сети, наполненная применимыми на практике знаниями, называется базой знаний (knowledge base).

bio-digital interface

Визуальная интерактивная среда, которая позволяет одновременно работать с базой знаний группе людей и цифровых устройств, объединённых в сеть.

knoxel

Способ отображения knyte'а в рамках био-цифрового интерфейса.

knoxel space

Способ отображения базы знаний, построенной по принципу унифицированной семантической сети.

unified bio-digital language

Универсальный способ коммуникации между людьми и цифровыми устройствами. Позволяет обмениваться данными, информацией и знаниями между людьми и цифровыми устройствами вне зависимости от их родного натурального языка и нативной системной архитектуры.
