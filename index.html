Базовые понятия цифровой инженерии знаний

В общем случае навряд ли можно измерить знания численно. Но если рассмотреть знания в контексте их конкретного цифрового представления, то можно выделить некоторые базовые информационные сущности, которые используются для описания знаний. Их размер и структура вполне могут быть определны.

knit

По аналогии с bit для измерения информации, knit - это наименьшая единица измерения знаний для цифрового устройства. Если каждую семантическую сущность обозначать через глобально уникальный идентификатор, то он и будет информационной структурой knit'а, а его размер - размером knit'а. Если использовать для идентфикации классический uuid v4. Тогда значение knit'а может выглядеть вот так:

ac2fb456-5936-48dc-934c-1bfcf6bd71e6
8a048d4c-93c0-45b8-8a36-e8a3cd80164d
71076d29-d4ff-430a-b9bd-9c9ee6e5584e

Таким образом размер knit'а равен

• 128 бит (16 байт) в бинарном представлении;
• 36 байт в текстовом представлении.

knyte

По аналогии с byte для измерения информации, knyte - это наименьшая единица измерения знаний для воспринятия человеком. Если byte - это количество bit, необходимых для кодирования одного символа в тексте, то knyte - это количество knit, необходимых для полного описания одной семантической сущности.

Предположим, что следующего описания семантической сущности будет достаточно для создания базы знаний:

core entity: entity id → knit → uuid v4
vector initial entity: entity id → knit → uuid v4
vector terminal entity: entity id → knit → uuid v4

Тогда размер knyte'а равен 3 knit'ам.

sizeof(knyte) === 3 * sizeof(knit)

information map

Для интерпретации человеком и для обработки цифровыми алгоритмами нужно иметь возможность привязывать к сущностям произвольную информацию. В общем случае информация может храниться в бинарной форме и измеряться в битах. В частном случае веб-реализации семантической сети информация будет храниться в текстовом виде и измеряться в байтах.

В качестве структуры данных для информационного контента семантических сущностей используется ассоциативный массив (hashmap). Для набора семантических сущностей информационная таблица может выглядеть так:

ac2fb456-5936-48dc-934c-1bfcf6bd71e6: "first entity"
8a048d4c-93c0-45b8-8a36-e8a3cd80164d: "100500"
71076d29-d4ff-430a-b9bd-9c9ee6e5584e: ""

knyte cloud

Структура данных, состоящая из массива связанных между собой knyte'ов, называется облаком knyte'ов. Предполагается что это универсальная структура для хранения семантической части базы знаний.

unified semantic web

Объединённые knyte cloud и её information map образуют унифицированную семантическую сеть (unified semantic web). Унифицированную, потому что сеть состоит из объектов одного типа - knyte'ов с привязанной к ним информацией. Семантическую, потому что её элементами являются семантические сущности. Сеть, потому что все объекты взаимосвязаны. В таком виде семантическая сеть имеет 2 размерности:

• семантический размер в knyte'ах, который вычисляется по облаку knyte'ов;
• информационный размер в byte'ах, который вычисляется по информационной таблице.

Итого описанная в примерах семантическая сеть имеет размер 3 knytes x 18 bytes.

knowledge base

Семантическая сеть, наполненная конкретными применимыми знаниями, называется базой знаний (knowledge base).

operational system

Операционная система (operational system) - конкретная реализация концепции унифицированной семантической сети, которая позволяет функционировать базе знаний в виде цифрового программно-аппаратного комплекса в сотрудничестве с человеком или группой людей.

bio-digital interface

Визуальная интерактивная среда, которая позволяет одновременно работать с базой знаний группе людей и группе цифровых устройств, объединённых в сеть.

knoxel

Способ отображения knyte'а в рамках био-цифрового интерфейса.

knoxel space

Способ отображения базы знаний, построенной по принципу унифицированной семантической сети.

unified bio-digital language

Универсальный способ коммуникаций между людьми и цифровыми устройствами. Позволяет обмениваться данными, информацией и знаниями между людьми и цифровыми устройствами вне зависимости от их родного натурального языка и системной архитектуры.
