Ревизия №7 от 2021.12.23

Мотивация

По нашим наблюдениям жизнь в современном обществе связана с постоянно возрастающими перегрузками восприятия, высоким интеллектуальным напряжением, необходимостью долго выискивать в потоке информационном шума крупицы полезного знания для выживания и самореализации. Создаётся ощущение, что развитие информационных технологий хоть и даёт нам больше функциональных возможностей, но всё сильнее нагружает психику потоками информации, которые приходится постоянно интерпретировть и фильтовать. Чтобы обеспечить собственное выживание человеку приходится следить за всё большим количеством виртуальных артефактов. Это обессиливает людей, лишает их радости жизни, возможности уделять достаточно внимания семье и самим себе.

Мы верим, что настало время развивать более высокоуровневую отрасль технологий для обуздания потоков информации, которые активно генерируются в процессе жизни нашего общества и каждого из нас. Инженерия знаний на основе метаязыка - это тот новый уровень контроля и понимания, который позволит осознать и оптимизировать как работу информационных технологий, так и обслуживаемые ими процессы жизнедеятельности: выживание, кооперация, образование, здравоохранение, трудовая деятельность, управление, производство и другое.

Цель

Построить эффективную архитектуру для взаимодействия людей и машин, ориентированную на оптимальное использование ментальных и природных ресурсов.

Основные этапы

  • Основы автоматизации работы со знаниями для облегчения общей ментальной нагрузки на творческих людей
  • Описание алгоритмов и структур данных через семантические структуры
  • Учёт семантических ресурсов, интегрированный в базовую архитектуру семантической сети
    • Автоматическое вознаграждение участников сети за вклад в создание и поддержание базы знаний и её инфраструктуры за счёт ресурсов самой сети
    • Предоставление услуг и поставка товаров в рамках сети в обмен на ресурсы самой сети
  • Поиск на основе знаний, описывающих структуру ожидаемого результата и методику поиска
    • Точный, быстрый и полный поиск необходимой информации в рамках всей глобальной семантической сети
    • Оценка спроса и предложений на рынке, быстрый поиск возможностей для обмена и кооперации
  • Самодостаточная экосистема людей, машин, ресурсов, данных, информации, знаний и, вероятно, чего-то ещё

Методика

В общем случае навряд ли можно измерить знания численно. Но если рассмотреть знания в контексте их конкретного цифрового представления, то можно выделить некоторые базовые структуры данных, которые могут быть использованы для описания знаний. Их конфигурация и размер вполне могут быть определены.

knit

По аналогии с bit для измерения информации, knit - это наименьшая единица измерения знаний для цифрового устройства. Если представить, что знания можно выразить совокупностью семантических сущностей, и каждую такую сущность обозначать через глобально уникальный идентификатор, то он и будет структурой данных для описания knit'а, а его размер - размером knit'а. Если использовать для идентфикации классический uuid v4, тогда значения knit'ов могут выглядеть вот так:

ac2fb456-5936-48dc-934c-1bfcf6bd71e6
8a048d4c-93c0-45b8-8a36-e8a3cd80164d
71076d29-d4ff-430a-b9bd-9c9ee6e5584e

Таким образом размер knit'а равен

  • 16 байт (128 бит) в бинарном представлении
  • 36 байт в текстовом представлении

knyte

По аналогии с byte для измерения информации, knyte - это наименьшая единица измерения знаний, доступных для воспринятия человеком. Если byte - это количество bit, необходимых для кодирования одного символа в тексте, то knyte - это количество knit, необходимых для полного описания одной абстрактной семантической сущности.

abstract semantic entity

Опишем абстрактную семантическую сущность следующей структурой данных:

core entity: entity id → knit → uuid v4
vector initial entity: entity id → knit → uuid v4
vector terminal entity: entity id → knit → uuid v4

Тогда размер абстрактной семантической сущности равен 3 knit'ам.

sizeof(abstractSemanticEntity) === 3 * sizeof(knit) // 48 байт в бинарном представлении

information map

Для интерпретации семантических сущностей и их сетей человеком, необходимо иметь возможность привязывать к ним информацию. Информация - это интерпретируемые данные. Объём этих данных измеряется в байтах. Данные могут интерпретироваться людьми как текстовая, визуальная, звуковая информация. Алгоритмы тоже могут интерпретировать данные (например, числовые, строковые, бинарные типы данных или составные структуры), работая на уровне информационных систем, а затем записывать результаты в виде семантических сущностей.

Если к абстрактной семантической сущности привязать информацию, то она становится обычной семантической сущностью. Её смысл можно в полной мере определить в контексте связей с другими семантическими сущностями.

В качестве структуры данных для описания информационной части семантических сущностей используется ассоциативный массив (hashmap). Для набора из 3 семантических сущностей информационная карта может выглядеть так:

ac2fb456-5936-48dc-934c-1bfcf6bd71e6: "first entity" // 12 байт
8a048d4c-93c0-45b8-8a36-e8a3cd80164d: "100500" // 6 байт
71076d29-d4ff-430a-b9bd-9c9ee6e5584e: "" // 0 байт

Объём данных в представленной информационной карте равен 18 байтам.

Каждый элемент в information map называется information record и принадлежит соответствующей семантической сущности.

single knyte

В итоге knyte - это объединение abstract semantic entity и соответствуюшей ему information record.

knyte cloud

Совокупность связанных между собой knyte'ов называется облаком knyte'ов. Данное понятие не предолагает визуализации, а относится к внутреннему представлению знаний внутри цифровой системы или в воображении человека.

В таком виде семантическая сеть имеет 2 размерности:

  • размер структуры в knyte'ах, который вычисляется по knyte cloud
  • объём информации в byte'ах, который вычисляется по information map

Описанный в примерах knyte cloud имеет размер 3 * sizeof(abstractSemanticEntity) + (12 + 6 + 0) // 66 байт.

semantic web

В общем понимании семантическая сеть - это способ представления знаний. Так как мы заинтересованы в работе с био-цифровой базой знаний, то требуем чтобы семантическая сеть была представлена в виде, одинаково понятном как для человека, так и для машины.

Исходя из описанной ранее модели, мы можем сказать, что knyte cloud - это гомогенная рекурсивная био-цифровая семантическая сеть.

  • Гомогенная, потому что она состоит из однотипных элементов - knyte'ов
  • Рекурсивная, потому что связью между двумя knyte'ами также является knyte.
  • Био-цифровая, потому что равно ориентирована как на людей, так и на машины.
  • Семантическая, потому что её элементами являются семантические сущности
  • Сеть, потому что элементы сети явным образом связаны между собой и не имеют связей ни на какие внешние объекты.
Для краткости далее в рамках наших исследований будет называть её просто семантической сетью, так как другие разновидности семантических сетей мы не будем рассматривать.

Отличие knyte cloud от классической семантической сети в том, что для задания отношений между понятиями используется не разновидность направленного математического графа, а гомогенный рекурсивный граф, в котором возможно задание отношений между отношениями. Визуально это соединение стрелками не только вершин, но и других стрелок. То есть один и тот же элемент рекурсивного графа может быть как вершиной в одной связке, так и ребром в другой связке. Это позволяет вводить все необходимые типы отношений на основе понятий самой семантической сети, а также использовать понятия в качестве отношений между другими понятиями. То есть в отличие от классической семантической сети, основанной на математическом графе, рекурсивный граф позволяет создавать самодостаточные семантические сети, которые полностью себя определяют. Это сложно понять теоретически, но становится абсолютно очевидным по мере использования интерактивного прототипа системы.

knowledge base

Конкретная программно-аппаратная реализация концепции семантической сети, наполненная применимыми на практике знаниями, называется базой знаний.

bio-digital interface

Био-цифровой интерфейс - это визуальная интерактивная среда, которая позволяет работать с базой знаний группе людей и цифровых устройств, объединённых в единую коммуникационную сеть.

knoxel

По аналогии с pixel в компьютерной графике, knoxel - это способ отображения knyte'а в рамках био-цифрового интерфейса.

knyte space

Способ отображения базы знаний, построенной по модели knyte cloud.

Прототип реализации данной идеи можно найти здесь. Внимательно читайте README.md)

bio-digital metalanguage

Био-цифровой метаязык - это универсальный способ коммуникации между людьми и цифровыми устройствами. Позволяет обмениваться данными, информацией и знаниями между людьми и цифровыми устройствами вне зависимости от их естественного языка и системной архитектуры. Как нам сейчас видится, он должен представлять собой базу знаний, может быть построен по прицнипам семантической сети в рамках прикладной модели knyte space.

Сам по себе метаязык - это наиболее масштабное из известных нам применение базы знаний на практике.
