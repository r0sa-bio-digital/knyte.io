Базовые понятия цифровой инженерии знаний

Ревизия №4 от 2020.04.23

Мотивация

По моим наблюдениям жизнь в современном обществе связана с постоянно возрастающими перегрузками восприятия, высоким интеллектуальным напряжением, необходимостью долго выискивать в потоке информационном шума крупицы полезного знания для выживания и самореализации. Создаётся ощущение, что развитие информационных технологий хоть и даёт нам больше функциональных возможностей, но всё сильнее нагружает психику потоками информации, которые приходится постоянно интерпретировть. Чтобы обеспечить собственное выживание человеку приходится следить всё за большим количеством виртуальных артефактов. Это обессиливает людей, лишает их радости жизни, возможности уделять достаточно внимания семье и самим себе.

Из-за непостоянства курсов валют, ненадёжности материальных ценностей и общественных договоров нет никаких гарантий, что накопленные в течение жизни активы не обесценятся. Это сильно понижает мотивацию делать для общества что-либо выходящее за рамки каждодневного выживания. Как следствие общество не развивается и не имеет запаса прочности перед лицом опасности или глобальной катастрофы.

Мне кажется, что необходимо развивать более высокоуровневую отрасль технологий для обуздания потоков информации, которые активно генерируются в процессе жизни нашего общества и каждого из нас. Инженерия знаний - это тот новый уровень контроля и понимания, который позволит осознать и оптимизировать как работу информационных технологий, так и обслуживаемые ими процессы жизнедеятельности: выживание, кооперация, образование, здравоохранение, трудовая деятельность, управление, производство и другое.

Цель

Найти способ взаимодействия людей и машин, направленный на эффективное и надёжное решение задач выживания, при котором остаётся свободное время и энергия на настоящую жизнь и развитие за пределами ограничений агрессивной среды, в которой мы родились и живём.

Основные этапы

  • Основы автоматизации работы со знаниями для облегчения общей ментальной нагрузки на творческих людей
  • Описание алгоритмов и структур данных через семантические структуры
  • Учёт семантических ресурсов, интегрированный в базовую архитектуру семантической сети
    • В каком-то смысле это аналог электронных денег, но не подверженный изменению ценности
    • Автоматическое вознаграждение за вклад в создание и поддержание инфраструктуры базы знаний
    • Предоставление услуг и поставка товаров за семантические ресурсы в рамках всей сети
  • Поиск на основе знаний, описывающих структуру ожидаемого результата и методику поиска
    • Точный, быстрый и полный поиск необходимой информации в рамках всей глобальной семантической сети
    • Оценка спроса и предложений на рынке, быстрый поиск возможностей для обмена и кооперации
  • Самодостаточная экосистема людей, машин, ресурсов, данных, информации, знаний и, вероятно, чего-то ещё

Методика

В общем случае навряд ли можно измерить знания численно. Но если рассмотреть знания в контексте их конкретного цифрового представления, то можно выделить некоторые базовые структуры данных, которые могут быть использованы для описания знаний. Их конфигурация и размер вполне могут быть определны.

knit

По аналогии с bit для измерения информации, knit - это наименьшая единица измерения знаний для цифрового устройства. Если представить, что знания можно выразить совокупностью семантических сущностей, и каждую такую сущность обозначать через глобально уникальный идентификатор, то он и будет структурой данных для описания knit'а, а его размер - размером knit'а. Если использовать для идентфикации классический uuid v4, тогда значения knit'ов могут выглядеть вот так:

ac2fb456-5936-48dc-934c-1bfcf6bd71e6
8a048d4c-93c0-45b8-8a36-e8a3cd80164d
71076d29-d4ff-430a-b9bd-9c9ee6e5584e

Таким образом размер knit'а равен

  • 128 бит (16 байт) в бинарном представлении
  • 36 байт в текстовом представлении

knyte

По аналогии с byte для измерения информации, knyte - это наименьшая единица измерения знаний, доступных для воспринятия человеком. Если byte - это количество bit, необходимых для кодирования одного символа в тексте, то knyte - это количество knit, необходимых для полного описания одной абстрактной семантической сущности.

Опишем абстрактную семантическую сущность следующей структурой данных:

core entity: entity id → knit → uuid v4
vector initial entity: entity id → knit → uuid v4
vector terminal entity: entity id → knit → uuid v4

Тогда размер knyte'а равен 3 knit'ам.

sizeof(knyte) === 3 * sizeof(knit) // 48 bytes

knyte cloud

Совокупность связанных между собой knyte'ов называется облаком knyte'ов. Мы исходим из предположения, что она может описывать абстрактные знания.

Что такое абстрактные знания? Это знания без конкретной интерпретации и применения. Некая обобщённая структура, выражающая суть такого явления как знания.

information map

Для интерпретации семантических сущностей и их сетей человеком или машиной, нужно иметь возможность привязывать к ним информацию. Информация - это интерпретируемые данные. Объём этих данных измеряется в байтах. Данные могут интерпретироваться как людьми (например, текстовая, визуальная, звуковая информация), так и алгоритмами (например, числовые, строковые, бинарные типы данных или составные структуры).

Если к абстрактной семантической сущности привязать информацию, то она становится обычной семантической сущностью. Её смысл можно интерпретировать в контексте связей с другими семантическими сущностями.

В качестве структуры данных для описания информационной части семантических сущностей используется ассоциативный массив (hashmap). Для набора из 3 семантических сущностей информационная карта может выглядеть так:

ac2fb456-5936-48dc-934c-1bfcf6bd71e6: "first entity" // 12 bytes
8a048d4c-93c0-45b8-8a36-e8a3cd80164d: "100500" // 6 bytes
71076d29-d4ff-430a-b9bd-9c9ee6e5584e: "" // 0 bytes

Объём данных в представленной информационной карте равен 18 байтам.

Каждый элемент в information map называется information record и принадлежит соответствующей семантической сущности.

unified semantic web

Объединённые knyte cloud и её information map образуют унифицированную семантическую сеть.

  • Унифицированная, потому что состоит из однотипных структур данных: knyte + information record
  • Семантическая, потому что её элементами являются семантические сущности
  • Сеть, потому что её объекты могут быть произвольно связаны друг с другом и не имеют связей ни на какие внешние объекты.
В таком виде семантическая сеть имеет 2 размерности:

  • размер структуры в knyte'ах, который вычисляется по knyte cloud
  • объём информации в byte'ах, который вычисляется по information map

Описанная в примерах семантическая сеть имеет размер 3 knytes x 18 bytes.

knowledge base

Конкретная программно-аппаратная реализация концепции унифицированной семантической сети, наполненная применимыми на практике знаниями, называется базой знаний.

bio-digital interface

Био-цифровой интерфейс - это визуальная интерактивная среда, которая позволяет работать с базой знаний группе людей и цифровых устройств, объединённых в коммуникационную сеть.

knoxel

Способ отображения knyte'а в рамках био-цифрового интерфейса.

knoxel space

Способ отображения базы знаний, построенной по принципу унифицированной семантической сети.

Прототип реализации данной идеи можно посмотреть здесь.

unified bio-digital language

Универсальный способ коммуникации между людьми и цифровыми устройствами. Позволяет обмениваться данными, информацией и знаниями между людьми и цифровыми устройствами вне зависимости от их натурального языка и системной архитектуры.
